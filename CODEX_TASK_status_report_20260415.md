**Проблема диагностирована:** Elder теряет на лонгах, выигрывает на шортах.

- Причина: BTC падает с пика $108k (дек 2025), шорты работают, лонги — нет.

**Решение добавлено в `_screen1_trend`:**
- Новый параметр `ETS2_TREND_REQUIRE_HIST_SIGN=1` — шорт разрешён только когда 4h MACD гистограмма **< 0** (подтверждённый даунтренд).

- Блокирует шорты в бычьем рынке (май-июн 2025 гистограмма положительная) → меньше ложных входов.

### 10. HZBO1 — alt_horizontal_break_v1 — НОВАЯ СТРАТЕГИЯ ✅

**Логика**: пробои ГОРИЗОНТАЛЬНЫХ зон поддержки/сопротивления.
Дополнение к ASB1 (который торгует наклонные линии):
- ASB1: входит когда НАКЛОННАЯ восходящая поддержка ломается
- HZBO1: входит когда ГОРИЗОНТАЛЬНАЯ зона поддержки ломается

**Зона поддержки** = кластер из минимум 3 swing lows в пределах 0.30 ATR друг от друга.
Пробой = цена закрывается НИЖЕ зоны на ≥ 0.40 ATR + медвежья свеча (body ≥ 45%) + объём ≥ 1.3× среднего.

**Ключевое открытие**: горизонтальные зоны существуют во ВСЕХ рыночных условиях (в отличие от наклонных линий), поэтому без строгой фильтрации — шум. Три обязательных фильтра:
1. `MIN_TOUCHES=3` — только сильные, многократно подтверждённые уровни
2. `VOL_MULT=1.3` — объём подтверждает пробой
3. `MACRO_REQUIRE_BEARISH=1` — шорт только при 4h MACD hist < 0

**WF-22 (шорты, selective config):**
```
AvgPF: 1.647  |  PF>1.0: 13/22 (59%)  |  PF>1.15: 13/22 (59%)
AvgTrades: ~10/window (низкая частота, высокое качество)
```

**Вердикт**: 🟢 VIABLE — деплой **0.40× риска** (пониженный риск из-за низкой частоты)
**Конфиг**: `configs/hzbo1_canary.env`

**Примечание**: низкая частота (10 сделок/45 дней) — стратегия "снайпер".
Стреляет только по самым чётким пробоям → высокий PF на каждую сделку.
Хорошее дополнение к портфелю, не перегружает позиции.
